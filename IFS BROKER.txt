                </div>
                <div class="requirement">
                    <h3>À propos de la norme IFS Broker</h3>
                    <p>IFS Broker est une norme de certification pour les entreprises qui agissent en tant qu'intermédiaires entre des fournisseurs et des clients dans la chaîne d'approvisionnement alimentaire et non-alimentaire. Cette norme s'applique aux sociétés qui n'ont pas de contact physique avec les produits mais qui sont légalement responsables des produits qu'elles commercialisent.</p>
                    <br>
                    <p>La certification IFS Broker permet de garantir que les courtiers mettent en œuvre des mesures adéquates pour assurer la sécurité et la qualité des produits qu'ils commercialisent, même s'ils ne manipulent pas physiquement ces produits.</p>
                </div>
                <div class="requirement">
                    <h3>Comment utiliser ce guide</h3>
                    <ul style="margin-left: 20px; margin-top: 10px;">
                        <li><strong>Navigation :</strong> Utilisez le menu de gauche pour accéder aux différentes sections et sous-sections de la norme.</li>
                        <li><strong>Recherche :</strong> Utilisez la barre de recherche pour trouver rapidement des exigences spécifiques.</li>
                        <li><strong>Détails des exigences :</strong> Cliquez sur une exigence pour afficher les questions d'audit, la documentation requise et les conseils pour les auditeurs.</li>
                        <li><strong>Points KO :</strong> Les exigences marquées "KO" (Knock Out) sont critiques. Une non-conformité sur ces points entraîne l'échec de l'audit.</li>
                    </ul>
                </div>
                <div class="requirement">
                    <h3>Points clés de la norme</h3>
                    <p>La norme IFS Broker version 3.2 comprend 8 points KO (Knock Out) qui sont essentiels pour la certification :</p>
                    <ul style="margin-left: 20px; margin-top: 10px;">
                        <li><strong>KO n°1 :</strong> Responsabilité de la direction (1.2.2)</li>
                        <li><strong>KO n°2 :</strong> Système de gestion des risques (2.3.1)</li>

                 <div class="section-header">
                    <h2>2.3 Système de gestion des risques</h2>
                    <p>Cette section concerne la mise en place d'un système de gestion des risques complet et systématique.</p>
                </div>
                <div class="requirement">
                    <div class="requirement-header" onclick="toggleRequirement(this)">
                        <span class="requirement-number">2.3.1</span>
                        <span class="requirement-title">La base du système de contrôle de la sécurité des produits doit être un système de gestion des risques</span>
                        <span class="ko-badge">KO n°2</span>
                    </div>
                    <div class="requirement-content">
                        <p>La base du système de contrôle de la sécurité des produits de l'entreprise doit être un système de gestion des risques entièrement mis en œuvre, systématique et complet.</p>
                        <div class="tabs">
                            <div class="tab active" onclick="changeTab(this, 'questions-2-3-1')">Questions d'audit</div>
                            <div class="tab" onclick="changeTab(this, 'advice-2-3-1')">Conseils pour les auditeurs</div>
                        </div>
                        <div class="tab-content active" id="questions-2-3-1">
                            <ul class="question-list">
                                <li>Le système de gestion des risques est établi selon quelles normes ?</li>
                                <li>Tous les risques du processus commercial/services de courtage sont-ils suffisamment pris en compte ?</li>
                            </ul>
                        </div>
                        <div class="tab-content" id="advice-2-3-1">
                            <p>Il doit y avoir un système de gestion des risques qui estime clairement et traite les risques existants du processus commercial et identifie les mesures de contrôle. Les risques typiques pour un courtier sont, par exemple, les dangers chez le producteur, les dangers pendant le transport, les dangers pendant le stockage (intermédiaire).</p>
                            <br>
                            <p>Si le courtier commercialise des produits de marque propre ou de marque client, il est responsable des produits. Par conséquent, l'évaluation des risques doit inclure les fournisseurs de produits ainsi que les fournisseurs de matières premières des fournisseurs (via le système de gestion des risques mis en œuvre par les fournisseurs de denrées alimentaires/autres produits).</p>
                            <br>
                            <p>Si le courtier commercialise des produits de marque fournisseur (par exemple, emballés, étiquetés standardisés), l'évaluation des risques peut se limiter aux fournisseurs de produits.</p>
                            <br>
                            <p>L'étude doit être basée sur les principes du Codex Alimentarius si des aliments sont commercialisés.</p>
                            <br>
                            <p><strong>Un KO serait donné si :</strong></p>
                            <ul class="advice-list">
                                <li>Aucune structure reconnaissable d'une évaluation des risques n'est disponible</li>
                                <li>L'évaluation des risques existante ne montre aucun lien avec les produits commercialisés</li>
                                <li>Des risques critiques n'ont pas été pris en compte</li>
                            </ul>
                            <p>Les aspects manquants dans l'évaluation des risques sont évalués, cependant, dans les exigences suivantes.</p>
                        </div>
                    </div>
                </div>
                <div class="requirement">
                    <div class="requirement-header" onclick="toggleRequirement(this)">
                        <span class="requirement-number">2.3.2</span>
                        <span class="requirement-title">L'entreprise doit s'assurer que le système de contrôle de la sécurité des produits de ses fournisseurs est un système de gestion des risques</span>
                    </div>
                    <div class="requirement-content">
                        <p>L'entreprise soumise à l'audit IFS Broker doit s'assurer que le système de contrôle de la sécurité des produits de ses fournisseurs est un système de gestion des risques entièrement mis en œuvre, systématique et complet. Il doit prendre en compte toutes les exigences légales des pays de production et de destination. Pour les aliments, un système HACCP est requis, basé sur les principes du Codex Alimentarius.</p>

                <div class="section-header">
                    <h2>4.1 Accord contractuel</h2>
                    <p>Cette section couvre les exigences liées aux accords contractuels entre les partenaires commerciaux.</p>
                </div>
                 <div class="requirement">
                    <div class="requirement-header" onclick="toggleRequirement(this)">
                        <span class="requirement-number">4.1.1</span>
                        <span class="requirement-title">Les exigences définies entre les partenaires contractuels doivent être établies, examinées et convenues</span>
                    </div>
                    <div class="requirement-content">
                        <p>Les exigences définies entre les partenaires contractuels doivent être établies, examinées et convenues concernant leur acceptabilité avant la conclusion d'un accord de fourniture. Toutes les clauses liées à la qualité et à la sécurité des produits doivent être communiquées et comprises par chaque employé concerné.</p>
                         <div class="tabs">
                            <div class="tab active" onclick="changeTab(this, 'questions-4-1-1')">Questions d'audit</div>
                            <div class="tab" onclick="changeTab(this, 'advice-4-1-1')">Conseils pour les auditeurs</div>
                        </div>
                        <div class="tab-content active" id="questions-4-1-1">
                            <ul class="question-list">
                                <li>Quelles réglementations contractuelles existent pour le(s) service(s) de courtage ?</li>
                                <li>Qui effectue la revue de contrat ? À quelle étape de la transaction commerciale la revue de contrat est-elle effectuée ?</li>
                                <li>Quels points sont examinés en détail ? Comment la revue effectuée est-elle enregistrée ?</li>
                                <li>Des critères de qualité ont-ils été déterminés ici ? Si oui, comment ces exigences sont-elles communiquées au sein du courtier ?</li>
                             </ul>
                        </div>
                        <div class="tab-content" id="advice-4-1-1">
                             <p>La revue de contrat est l'exigence la plus importante pour les courtiers. Ils doivent examiner en détail si les exigences de leurs clients peuvent être satisfaites par eux-mêmes et leurs fournisseurs. Ici, l'auditeur doit vérifier quelles exigences client existent et comment elles sont transmises aux fournisseurs respectifs. Dans de nombreux cas, le client ne connaît pas le fabricant et vice versa, car le courtier garde ces informations pour lui pour des raisons de compétitivité. Ensuite, le courtier doit convertir toutes les exigences client (par exemple, spécifications, termes et conditions) en ses propres documents et, si nécessaire, les envoyer aux fournisseurs éligibles. C'est souvent une source d'erreurs, surtout si le client modifie les spécifications lors de transactions en cours et que ces informations ne sont pas transmises aux fournisseurs du tout ou dans un délai opportun (voir aussi 4.1.3).</p>
                        </div>
                    </div>
                </div>
                 <div class="requirement">
                     <div class="requirement-header" onclick="toggleRequirement(this)">
                        <span class="requirement-number">4.1.2</span>
                        <span class="requirement-title">Les modifications des accords contractuels existants doivent être documentées et communiquées</span>
                    </div>
                    <div class="requirement-content">
                         <p>Les modifications des accords contractuels existants doivent être documentées et communiquées entre les partenaires contractuels.</p>
                         <div class="tabs">
                            <div class="tab active" onclick="changeTab(this, 'questions-4-1-2')">Questions d'audit</div>
                            <div class="tab" onclick="changeTab(this, 'advice-4-1-2')">Conseils pour les auditeurs</div>
                        </div>
                         <div class="tab-content active" id="questions-4-1-2">
                            <ul class="question-list">
                                <li>À quelle fréquence les modifications de commande se produisent-elles en pratique ?</li>
                                <li>Comment sont-elles enregistrées ?</li>
                                <li>Comment les fournisseurs et les propres employés sont-ils informés des modifications de commande des clients ?</li>
                                <li>Comment les clients sont-ils informés ?</li>
                            </ul>
                        </div>
                        <div class="tab-content" id="advice-4-1-2">
                             <p>Existe-t-il des réglementations concernant les modifications de commande dans la documentation ?</p>
                        </div>
                    </div>
                 </div>
                 <div class="requirement">
                     <div class="requirement-header" onclick="toggleRequirement(this)">
                        <span class="requirement-number">4.1.3</span>
                        <span class="requirement-title">Les exigences spécifiques de qualité et de sécurité des clients doivent être communiquées</span>
                    </div>
                    <div class="requirement-content">
                        <p>Les exigences spécifiques de qualité et de sécurité des clients doivent être communiquées et comprises par le fournisseur et/ou le prestataire de services de l'entreprise.</p>
                         <div class="tabs">
                            <div class="tab active" onclick="changeTab(this, 'questions-4-1-3')">Questions d'audit</div>
                            <div class="tab" onclick="changeTab(this, 'advice-4-1-3')">Conseils pour les auditeurs</div>
                        </div>
                         <div class="tab-content active" id="questions-4-1-3">
                            <ul class="question-list">
                                <li>Y a-t-il des exigences spécifiques des clients ?</li>
                                <li>Le courtier a-t-il clairement déterminé les responsabilités et les procédures pour examiner les exigences client ?</li>
                                <li>Si oui, comment sont-elles transmises aux fournisseurs ?</li>
                                <li>Comment est-il possible de vérifier ultérieurement la transmission aux fournisseurs ?</li>
                            </ul>
                        </div>
                        <div class="tab-content" id="advice-4-1-3">
                            <p>Il est important de vérifier ici si toutes les exigences produit d'un client sont réellement transmises par le courtier au fournisseur/fabricant. Souvent, le courtier prépare ses propres spécifications qui sont transmises aux fournisseurs. Ici, une comparaison sur la base des activités commerciales réalisées est requise pour tous les produits ou groupes de produits importants. Exemples d'exigences spécifiques de qualité et de sécurité client : nécessité de mettre en œuvre des détecteurs de métaux, bio, sans OGM, identité préservée, etc.</p>
                        </div>
                    </div>
                 </div>

             <div class="section" id="section4-2">
                 <div class="section-header">
                    <h2>4.2 Spécifications</h2>
                    <p>Cette section concerne les spécifications des produits commercialisés par le courtier.</p>
                </div>
                <div class="requirement">
                    <div class="requirement-header" onclick="toggleRequirement(this)">
                        <span class="requirement-number">4.2.1</span>
                        <span class="requirement-title">Des spécifications doivent être disponibles et en place pour tous les produits</span>
                    </div>
                    <div class="requirement-content">
                        <p>Elles doivent être à jour, sans ambiguïté et conformes aux exigences légales du ou des pays de destination et également aux exigences des clients.</p>
                        <div class="tabs">
                            <div class="tab active" onclick="changeTab(this, 'questions-4-2-1')">Questions d'audit</div>
                            <div class="tab" onclick="changeTab(this, 'advice-4-2-1')">Conseils pour les auditeurs</div>
                        </div>
                        <div class="tab-content active" id="questions-4-2-1">
                            <ul class="question-list">
                                <li>Existe-t-il des règles pour le contrôle des spécifications (préparation, examen, approbation, distribution) ?</li>
                                <li>Quel contenu minimal a été déterminé pour les spécifications ?</li>
                                <li>Comment les produits de marque fournisseur sont-ils spécifiés ?</li>
                                <li>Si des spécifications sont disponibles : qui les examine et s'assure qu'elles sont à jour ?</li>
                                <li>Les réglementations légales de tous les pays de destination sont-elles prises en compte ?</li>
                            </ul>
                        </div>
                        <div class="tab-content" id="advice-4-2-1">
                            <p>Les spécifications pour les produits de marque A (type bien établi de produits de marque fournisseur dans l'emballage du consommateur final) ne sont généralement pas disponibles. Des informations sur des produits spécifiques (par exemple, tailles d'unité, exigences de température) sont toutefois requises.</p>
                            <p>Le secteur des fruits et légumes commercialise souvent sur la base des normes de commercialisation de l'UE (UNECE). Les spécifications sont alors inutiles sauf si le client l'exige. Il existe, cependant, presque toujours des spécifications spécifiques disponibles pour les produits de marque client (par exemple, marque de distributeur).</p>
                        </div>
                    </div>
                </div>
                <div class="requirement">
                    <div class="requirement-header" onclick="toggleRequirement(this)">
                        <span class="requirement-number">4.2.2</span>
                        <span class="requirement-title">Les spécifications du client doivent être respectées</span>
                        <span class="ko-badge">KO n°3</span>
                    </div>
                    <div class="requirement-content">
                        <p>Les spécifications du client doivent être respectées.</p>
                        <div class="tabs">
                            <div class="tab active" onclick="changeTab(this, 'questions-4-2-2')">Questions d'audit</div>
                            <div class="tab" onclick="changeTab(this, 'advice-4-2-2')">Conseils pour les auditeurs</div>
                        </div>
                        <div class="tab-content active" id="questions-4-2-2">
                            <ul class="question-list">
                                <li>Comment le courtier s'assure-t-il que les exigences client, qui constituent la base de la transaction commerciale, sont transmises intégralement au fournisseur/fabricant ?</li>
                                <li>Quels sont les paramètres clés dans les spécifications client ?</li>
                                <li>Comment le courtier vérifie-t-il que les spécifications client sont entièrement respectées (par exemple, analyses en laboratoire, inspections des marchandises effectuées par des prestataires de services au port) ?</li>
                            </ul>
                        </div>
                        <div class="tab-content" id="advice-4-2-2">
                            <p><strong>Un KO doit être attribué si :</strong> des aspects essentiels de la qualité de la spécification n'ont pas été respectés. C'est toujours le cas si les paramètres spécifiés diffèrent significativement.</p>
                            <br>
                            <p><strong>Exemples de KO :</strong></p>
                            <ul class="advice-list">
                                <li>Pays d'origine incorrect</li>
                                <li>Composition des portions spécifiées non conforme</li>
                                <li>Absence de substances indésirables spécifiées par le client manquante dans la spécification du fournisseur</li>
                            </ul>
                            <br>
                            <p>Le KO ne sera pas donné ici si l'inspection des marchandises livrées montre que, bien qu'elles s'écartent de la spécification du client, le courtier a convenu de la spécification correcte avec le fournisseur. Ce cas est évalué selon d'autres exigences, par exemple évaluation du fournisseur, inspections, produits non conformes, actions correctives.</p>
                        </div>
                    </div>
                </div>
                <div class="requirement">
                     <div class="requirement-header" onclick="toggleRequirement(this)">
                        <span class="requirement-number">4.2.3</span>
                        <span class="requirement-title">Si requis par les clients, les spécifications produit doivent être formellement convenues</span>
                    </div>
                    <div class="requirement-content">
                         <p>Si requis par les clients, les spécifications produit doivent être formellement convenues.</p>
                         <div class="tabs">
                            <div class="tab active" onclick="changeTab(this, 'questions-4-2-3')">Questions d'audit</div>
                         </div>
                         <div class="tab-content active" id="questions-4-2-3">
                            <ul class="question-list">
                                <li>Les clients exigent-ils un accord formel sur les spécifications produit ? Si oui, quels produits sont concernés ?</li>
                            </ul>
                        </div>
                    </div>
                 </div>
                <div class="requirement">
                    <div class="requirement-header" onclick="toggleRequirement(this)">
                        <span class="requirement-number">4.2.4</span>
                        <span class="requirement-title">Une procédure doit exister pour la création, la modification et l'approbation des spécifications</span>
                    </div>
                    <div class="requirement-content">
                        <p>Une procédure doit exister pour la création, la modification et l'approbation des spécifications pour tous les produits et parties des services, qui doit inclure l'acceptation préliminaire du client, si des spécifications ont été convenues avec les clients.</p>
                        <div class="tabs">
                            <div class="tab active" onclick="changeTab(this, 'questions-4-2-4')">Questions d'audit</div>
                            <div class="tab" onclick="changeTab(this, 'advice-4-2-4')">Conseils pour les auditeurs</div>
                        </div>
                        <div class="tab-content active" id="questions-4-2-4">
                            <ul class="question-list">
                                <li>Quelles spécifications sont gérées par le courtier (par exemple, aperçu) ?</li>
                                <li>Quels types de déterminations existent pour la préparation, l'examen et l'approbation des spécifications ?</li>
                                <li>Quelles réglementations sont définies pour la transmission des informations des spécifications aux fournisseurs et aux clients ?</li>
                             </ul>
                        </div>
                         <div class="tab-content" id="advice-4-2-4">

            <div class="section" id="section4-3">
                 <div class="section-header">
                    <h2>4.3 Développement de produit / Modification de produit / Modification des processus de production</h2>
                    <p>Cette section s'applique aux courtiers impliqués dans le développement de produits ou la modification de produits existants.</p>
                    <div class="note"><strong>Note:</strong> Ces exigences peuvent être évaluées comme N/A (Non Applicable) en cas de produits de marque fournisseur. Mais dès que le courtier est considéré comme le premier metteur sur le marché (produits importés), qu'il a ses propres produits de marque (Broker) ou qu'il développe des produits de marque client, ce chapitre s'applique.</div>
                </div>
                <div class="requirement">
                    <div class="requirement-header" onclick="toggleRequirement(this)">
                        <span class="requirement-number">4.3.1</span>
                        <span class="requirement-title">Une procédure de développement produit doit être en place</span>
                    </div>
                    <div class="requirement-content">
                        <p>Une procédure de développement produit doit être en place pour tous les produits de marque propre et client, qui prend en compte les principes d'évaluation des risques (et le système HACCP, selon le Codex Alimentarius, pour les produits alimentaires), y compris la fraude alimentaire. La procédure doit garantir que tous les produits existants et nouveaux sont conçus pour répondre aux exigences légales et client. Cette procédure doit également tenir compte des brevets, le cas échéant.</p>
                        <div class="tabs">
                            <div class="tab active" onclick="changeTab(this, 'questions-4-3-1')">Questions d'audit</div>
                            <div class="tab" onclick="changeTab(this, 'advice-4-3-1')">Conseils pour les auditeurs</div>
                        </div>
                        <div class="tab-content active" id="questions-4-3-1">
                            <ul class="question-list">
                                <li>Le courtier effectue-t-il des activités de développement ? Si oui, quel type de développements ?</li>
                                <li>Existe-t-il une procédure indiquant le processus de développement ? Si oui, comment la conformité aux exigences légales et client est-elle assurée pour le développement ?</li>
                                <li>Existe-t-il une planification du développement ?</li>
                            </ul>
                        </div>
                        <div class="tab-content" id="advice-4-3-1">
                            <p>Si un courtier effectue des développements, cela ne peut être réalisé qu'en très étroite coopération avec des entreprises/fabricants sélectionnés. Ici, l'auditeur doit vérifier de quelle manière la mise en œuvre pratique est réalisée et comment le courtier vérifie ou accompagne la conformité avec toutes les étapes de développement pertinentes et les spécifications produit chez le fabricant sélectionné.</p>
                        </div>
                    </div>
                </div>
                <div class="requirement">
                    <div class="requirement-header" onclick="toggleRequirement(this)">
                        <span class="requirement-number">4.3.2</span>
                        <span class="requirement-title">L'entreprise doit assumer la responsabilité de la formulation, des processus et des paramètres</span>
                    </div>
                    <div class="requirement-content">
                        <p>L'entreprise doit assumer la responsabilité de la formulation du produit, des processus de fabrication, des paramètres de processus et de la réalisation des exigences produit. Les paramètres ci-dessus doivent être établis et garantis par des essais en usine et/ou des tests de produits.</p>
                        <div class="tabs">
                            <div class="tab active" onclick="changeTab(this, 'questions-4-3-2')">Questions d'audit</div>
                        </div>
                        <div class="tab-content active" id="questions-4-3-2">
                             <ul class="question-list">
                                <li>Quels enregistrements concernant les essais de production sont conservés par les entreprises contractantes ?</li>
                                <li>Comment le fournisseur informe-t-il le courtier des résultats des essais ?</li>
                                <li>Des tests requis pendant un développement sont-ils déterminés sur la base des spécifications ? Comment est-il possible de vérifier le résultat ?</li>
                                <li>Le laboratoire propre et/ou un laboratoire externe accrédité effectue-t-il l'analyse requise ? Qui sélectionne le laboratoire ?</li>
                                <li>Le courtier accompagne-t-il les essais de production réalisés dans les entreprises ? Si oui, quels propres enregistrements en plus de ceux du fournisseur sont conservés ici ?</li>
                             </ul>
                        </div>
                    </div>
                </div>
                <div class="requirement">
                    <div class="requirement-header" onclick="toggleRequirement(this)">
                        <span class="requirement-number">4.3.3</span>
                        <span class="requirement-title">L'entreprise doit s'assurer que des tests de durée de vie sont effectués le cas échéant</span>
                    </div>
                    <div class="requirement-content">
                        <p>Le cas échéant, l'entreprise doit s'assurer que des tests de durée de vie ou des processus adéquats ont été effectués et que la formulation du produit, l'emballage, la fabrication et les conditions déclarées ont été pris en compte, pour établir la durabilité minimale du produit.</p>
                         <div class="tabs">
                            <div class="tab active" onclick="changeTab(this, 'questions-4-3-3')">Questions d'audit</div>
                        </div>
                        <div class="tab-content active" id="questions-4-3-3">
                            <ul class="question-list">
                                <li>Comment la durée de vie des produits est-elle déterminée ? Y a-t-il des spécifications client pour cela ?</li>
                                <li>Comment les tests de durée de vie sont-ils effectués ? Quels paramètres sont examinés ?</li>
                                <li>Comment la durée de vie est-elle déterminée si les tests de durée de vie ne sont pas effectués ?</li>
                            </ul>
                        </div>
                    </div>
                </div>
                <div class="requirement">
                     <div class="requirement-header" onclick="toggleRequirement(this)">
                        <span class="requirement-number">4.3.4</span>
                        <span class="requirement-title">En relation avec le développement de produits alimentaires, des évaluations organoleptiques doivent être entreprises</span>
                    </div>
                    <div class="requirement-content">
                         <p>En relation avec le développement de produits alimentaires, l'entreprise doit s'assurer que des évaluations organoleptiques sont entreprises et que les résultats de ces évaluations sont examinés et pris en compte.</p>
                        <div class="tabs">
                            <div class="tab active" onclick="changeTab(this, 'questions-4-3-4')">Questions d'audit</div>
                        </div>
                         <div class="tab-content active" id="questions-4-3-4">
                            <ul class="question-list">
                                <li>Quelles caractéristiques organoleptiques sont pertinentes pour le produit à développer ?</li>
                                <li>Quels tests organoleptiques sont effectués ? Qui fait partie de l'équipe de dégustation ?</li>
                                <li>Les tests organoleptiques sont-ils également effectués par un laboratoire externe ?</li>
                                <li>Le client est-il impliqué dans les tests organoleptiques ?</li>
                             </ul>
                        </div>
                    </div>
                </div>
                <div class="requirement">
                     <div class="requirement-header" onclick="toggleRequirement(this)">
                        <span class="requirement-number">4.3.5</span>
                        <span class="requirement-title">Un processus doit être en place pour assurer la conformité de l'étiquetage</span>
                    </div>
                    <div class="requirement-content">
                        <p>Un processus doit être en place pour garantir que l'étiquetage de tous les produits existants et nouveaux est conforme à la législation en vigueur du pays de destination et aux exigences client.</p>
                        <div class="tabs">
                            <div class="tab active" onclick="changeTab(this, 'questions-4-3-5')">Questions d'audit</div>
                        </div>
                         <div class="tab-content active" id="questions-4-3-5">
                             <ul class="question-list">
                                <li>Qui vérifie l'étiquetage/la déclaration du nouveau produit ?</li>
                                <li>Comment la légalité est-elle assurée ?</li>
                                <li>Les produits sont livrés dans quels pays ? Comment le courtier assure-t-il la déclaration correcte pour tous les pays de destination ?</li>
                                <li>Existe-t-il des spécifications client liées à l'étiquetage/la déclaration ?</li>
                                <li>Quelle est la forme des spécifications pour les vérifications de déclaration déterminée dans la documentation du courtier ?</li>
                            </ul>
                        </div>
                    </div>
                </div>
                <div class="requirement">
                     <div class="requirement-header" onclick="toggleRequirement(this)">
                        <span class="requirement-number">4.3.6</span>
                        <span class="requirement-title">Des recommandations pour la préparation/utilisation doivent être établies</span>
                    </div>
                    <div class="requirement-content">
                        <p>Des recommandations pour la préparation et/ou l'utilisation des produits doivent être établies. Le cas échéant, les recommandations d'utilisation doivent concerner la satisfaction et la sécurité du consommateur. Si spécifié, les exigences client doivent être incluses.</p>
                        <div class="tabs">
                            <div class="tab active" onclick="changeTab(this, 'questions-4-3-6')">Questions d'audit</div>
                        </div>
                        <div class="tab-content active" id="questions-4-3-6">
                             <ul class="question-list">
                                <li>Existe-t-il des recommandations pour la préparation et/ou l'utilisation des produits ? Si oui, comment sont-elles coordonnées avec les exigences client ?</li>
                                <li>Comment l'exactitude des recommandations pour la préparation et/ou l'utilisation des produits est-elle vérifiée ?</li>
                            </ul>
                        </div>
                    </div>
                </div>
                <div class="requirement">
                     <div class="requirement-header" onclick="toggleRequirement(this)">
                        <span class="requirement-number">4.3.7</span>
                        <span class="requirement-title">Si le développement produit est prédéfini par le client, l'entreprise doit s'assurer que toutes les exigences sont respectées</span>
                    </div>
                    <div class="requirement-content">
                        <p>Si le développement produit est prédéfini par le client, l'entreprise doit s'assurer que toutes les exigences produit définies sont respectées.</p>
                        <div class="tabs">
                            <div class="tab active" onclick="changeTab(this, 'questions-4-3-7')">Questions d'audit</div>
                            <div class="tab" onclick="changeTab(this, 'documentation-4-3-7')">Documentation</div>
                        </div>
                        <div class="tab-content active" id="questions-4-3-7">
                             <ul class="question-list">
                                <li>Un client a-t-il des exigences spécifiques sur les procédures et/ou les étapes de développement ?</li>
                                <li>Comment le courtier s'assure-t-il que ces exigences sont respectées ?</li>
                             </ul>
                        </div>
                        <div class="tab-content" id="documentation-4-3-7">
                             <ul class="documentation-list">
                                <li>Spécification client</li>
                                <li>Enregistrements de vérification</li>
                            </ul>
                        </div>
                    </div>
                </div>
                <div class="requirement">
                    <div class="requirement-header" onclick="toggleRequirement(this)">
                        <span class="requirement-number">4.3.8</span>
                        <span class="requirement-title">L'avancement et les résultats du développement produit doivent être correctement enregistrés</span>
                    </div>
                    <div class="requirement-content">
                         <p>L'avancement et les résultats du développement produit doivent être correctement enregistrés. Les enregistrements pertinents pour la sécurité, la légalité et la qualité du produit doivent être disponibles dans l'entreprise.</p>
                         <div class="tabs">
                            <div class="tab active" onclick="changeTab(this, 'questions-4-3-8')">Questions d'audit</div>
                            <div class="tab" onclick="changeTab(this, 'advice-4-3-8')">Conseils pour les auditeurs</div>
                        </div>
                         <div class="tab-content active" id="questions-4-3-8">
                             <ul class="question-list">
                                <li>Quels enregistrements et résultats le courtier a-t-il identifiés comme pertinents pour la sécurité, la légalité et la qualité du produit ?</li>
                                <li>Quels enregistrements sont conservés et classés par le courtier concernant le travail de développement effectué ?</li>
                                <li>Quels enregistrements de développement l'entreprise exécutante conserve-t-elle ?</li>
                            </ul>
                        </div>
                         <div class="tab-content" id="advice-4-3-8">
                             <p>Habituellement, tous les enregistrements de développement sont disponibles dans cette entreprise/fournisseur qui a développé le produit pour le compte du courtier. Pour le cas individuel, l'auditeur doit vérifier quels enregistrements les courtiers doivent avoir au bureau afin que toutes les étapes du développement, dont ils sont responsables, puissent être tracées. Plus le courtier est activement intégré dans les processus de développement, plus les enregistrements détaillés sont à prévoir. Le courtier doit présenter des enregistrements fournissant des informations sur la légalité et les processus importants (par exemple, vérification de la législation alimentaire, HACCP ou sécurité de l'équipement pour les non-alimentaires).</p>
                        </div>
                    </div>
                </div>

             <div class="section" id="section4-4">
                 <div class="section-header">
                    <h2>4.4 Achats</h2>
                    <p>Cette section détaille les exigences relatives au processus d'achat et à la gestion des fournisseurs.</p>
                     <div class="note"><strong>Note:</strong> ce chapitre est toujours applicable pour un courtier.</div>
                 </div>
                <div class="requirement">
                     <div class="requirement-header" onclick="toggleRequirement(this)">
                        <span class="requirement-number">4.4.1</span>
                        <span class="requirement-title">L'entreprise doit contrôler les processus d'achat</span>
                    </div>
                    <div class="requirement-content">
                        <p>L'entreprise doit contrôler les processus d'achat pour s'assurer que tous les produits et services achetés, qui ont un impact sur la sécurité et la qualité des produits, sont conformes aux exigences.</p>
                         <div class="tabs">
                            <div class="tab active" onclick="changeTab(this, 'questions-4-4-1')">Questions d'audit</div>
                            <div class="tab" onclick="changeTab(this, 'advice-4-4-1')">Conseils pour les auditeurs</div>
                        </div>
                         <div class="tab-content active" id="questions-4-4-1">
                            <ul class="question-list">
                                <li>Les transactions d'achat sont basées sur quels critères ?</li>
                                <li>Comment les exigences client sont-elles intégrées dans les spécifications d'achat ?</li>
                                <li>Y a-t-il des spécifications fondamentales du client pour les sites de production des fournisseurs (par exemple, certificats) ? Comment sont-elles prises en compte ?</li>
                                <li>Quelles spécifications du client existent pour les prestataires de services contractuels ? Le service des achats les transmet-il au fournisseur ?</li>
                            </ul>
                        </div>
                        <div class="tab-content" id="advice-4-4-1">
                            <p>Les ventes et les achats chez un courtier doivent toujours travailler en étroite collaboration ; souvent l'acheteur et le vendeur sont la même personne. Le client peut spécifier la nature des marchandises par le biais de ses exigences produit (par exemple, spécification).</p>
                        </div>
                    </div>
                </div>
                <div class="requirement">
                     <div class="requirement-header" onclick="toggleRequirement(this)">
                        <span class="requirement-number">4.4.2</span>
                        <span class="requirement-title">Une procédure d'approbation et de suivi des fournisseurs et prestataires de services doit exister</span>
                    </div>
                    <div class="requirement-content">
                        <p>Une procédure d'approbation et de suivi des fournisseurs et prestataires de services doit exister.</p>
                        <div class="tabs">
                            <div class="tab active" onclick="changeTab(this, 'questions-4-4-2')">Questions d'audit</div>
                            <div class="tab" onclick="changeTab(this, 'advice-4-4-2')">Conseils pour les auditeurs</div>
                        </div>
                        <div class="tab-content active" id="questions-4-4-2">
                            <ul class="question-list">
                                <li>Quels sont les prérequis que les fournisseurs/prestataires de services doivent remplir avant d'être autorisés à livrer ?</li>
                                <li>Comment le courtier informe-t-il les fournisseurs sur les exigences d'approbation ?</li>
                                <li>Quelles déterminations pour l'évaluation des fournisseurs ont été faites ?</li>
                                <li>Quels fournisseurs sont évalués ?</li>
                                <li>Comment le courtier gère-t-il les fournisseurs bloqués et s'assure-t-il qu'aucune marchandise n'est achetée auprès d'eux ?</li>
                            </ul>
                        </div>
                        <div class="tab-content" id="advice-4-4-2">
                             <p>La procédure d'approbation et de suivi des fournisseurs devrait inclure au minimum les critères suivants : conformité aux spécifications, analyse des réclamations survenues, fiabilité de livraison, conformité aux exigences légales. Établir des critères supplémentaires est judicieux et doit être déterminé individuellement.</p>
                        </div>
                    </div>
                </div>
                <div class="requirement">
                    <div class="requirement-header" onclick="toggleRequirement(this)">
                        <span class="requirement-number">4.4.3</span>
                        <span class="requirement-title">La procédure d'approbation et de suivi doit être basée sur l'analyse des dangers et l'évaluation des risques</span>
                    </div>
                    <div class="requirement-content">
                         <p>La procédure d'approbation et de suivi doit être basée sur l'analyse des dangers et l'évaluation des risques associés et doit contenir des critères d'évaluation clairs tels que :</p>
                        <ul>
                            <li>audits,</li>
                            <li>certificats d'analyse,</li>
                            <li>fiabilité du fournisseur et réclamations (y compris la fraude), ainsi que</li>
                            <li>normes de performance requises</li>
                         </ul>
                        <div class="tabs">
                            <div class="tab active" onclick="changeTab(this, 'questions-4-4-3')">Questions d'audit</div>
                            <div class="tab" onclick="changeTab(this, 'advice-4-4-3')">Conseils pour les auditeurs</div>
                        </div>
                        <div class="tab-content active" id="questions-4-4-3">
                            <ul class="question-list">
                                <li>Quels critères sont pris en considération pour le processus d'approbation des fournisseurs ?</li>
                                <li>Le fournisseur est-il certifié selon l'un d'eux ?</li>
                                <li>Le courtier demande-t-il le renouvellement des certificats fournisseurs sur une base régulière ?</li>
                                <li>Le courtier effectue-t-il des audits fournisseurs (seconde partie) ? Si oui, comment sont-ils enregistrés ? Y a-t-il un plan d'action issu des audits fournisseurs ? Qui effectue les audits fournisseurs (membres du personnel interne/prestataires de services) ? Comment les auditeurs ont-ils été qualifiés ?</li>
                                <li>Les audits fournisseurs sont-ils effectués par des tiers contractuels ? Quels critères sont inspectés par le prestataire de services ?</li>
                             </ul>
                        </div>
                        <div class="tab-content" id="advice-4-4-3">
                             <p>Ici, il est important de vérifier quelles spécifications client sont mises en œuvre pour chaque fournisseur.</p>
                             <p>Conseils pour les courtiers de fruits & légumes : dans le secteur des fruits et légumes, la vente au détail demande des normes de producteurs. Celles-ci sont complétées par des exigences spécifiques pour les niveaux approuvés de résidus qui sont également convenus par contrat.</p>
                        </div>
                    </div>
                </div>
                 <div class="requirement">
                     <div class="requirement-header" onclick="toggleRequirement(this)">
                        <span class="requirement-number">4.4.4</span>
                        <span class="requirement-title">Le fournisseur du produit doit être certifié selon une norme IFS ou une autre norme reconnue par GFSI</span>
                    </div>
                    <div class="requirement-content">
                         <p>Le fournisseur du produit doit être certifié selon une norme IFS ou toute autre norme reconnue par GFSI couvrant le champ d'activité respectif. Des exceptions ne peuvent être faites que si le client accepte expressément d'autres conditions.</p>
                        <div class="tabs">
                            <div class="tab active" onclick="changeTab(this, 'questions-4-4-4')">Questions d'audit</div>
                            <div class="tab" onclick="changeTab(this, 'advice-4-4-4')">Conseils pour les auditeurs</div>
                        </div>
                        <div class="tab-content active" id="questions-4-4-4">
                             <ul class="question-list">
                                <li>Est-il possible d'appliquer une norme IFS (IFS Food, IFS HPC, IFS PACsecure, IFS Broker) ou le niveau intermédiaire IFS Progress aux produits échangés ?</li>
                                <li>Est-il possible d'appliquer une norme reconnue par GFSI incluant le bon champ d'application aux produits échangés ?</li>
                                <li>Quelles spécifications client existent concernant les certifications des fournisseurs ?</li>
                                <li>Existe-t-il un aperçu transparent des certificats existants des fournisseurs (par exemple, IFS, BRC, GlobalGAP, SQF) ?</li>
                                <li>En cas de questions concernant l'interprétation des normes et programmes IFS, veuillez contacter standardmanagement@ifs-certification.com</li>
                            </ul>
                        </div>
                         <div class="tab-content" id="advice-4-4-4">
                            <p>Une liste des normes actuellement comparées par GFSI peut être trouvée ici : https://mygfsi.com/how-to-implement/recognition/</p>
                            <p>Veuillez noter que certains schémas de certification régionaux ne sont pas reconnus ou – selon le produit – les schémas n'ont que des champs d'application spécifiques.</p>
                            <p>Les évaluations IFS Progress ne sont pas considérées comme "Standard" ou "certifiées".</p>
                        </div>
                    </div>
                 </div>
                 <div class="requirement">
                     <div class="requirement-header" onclick="toggleRequirement(this)">
                        <span class="requirement-number">4.4.5</span>
                        <span class="requirement-title">L'entreprise doit avoir un système basé sur les risques pour surveiller les zones d'approvisionnement</span>
                    </div>
                    <div class="requirement-content">
                        <p>L'entreprise doit avoir un système (interne ou externe) basé sur les risques en place, pour surveiller les zones d'approvisionnement des produits achetés.</p>
                        <div class="tabs">
                            <div class="tab active" onclick="changeTab(this, 'questions-4-4-5')">Questions d'audit</div>
                        </div>
                         <div class="tab-content active" id="questions-4-4-5">
                             <ul class="question-list">
                                <li>Le courtier/client a-t-il spécifié les régions d'origine des produits ?</li>
                                <li>Le courtier connaît-il l'origine de tous les produits échangés et y a-t-il des preuves pour chaque livraison individuelle ?</li>
                                <li>Comment l'origine des produits achetés est-elle surveillée ? Un système a-t-il été mis en place pour cela ? Le courtier s'appuie-t-il sur des outils/sources déjà existants ?</li>
                             </ul>
                        </div>
                    </div>
                 </div>
                 <div class="requirement">
                    <div class="requirement-header" onclick="toggleRequirement(this)">
                        <span class="requirement-number">4.4.6</span>
                        <span class="requirement-title">Une évaluation des fournisseurs et prestataires de services doit être faite régulièrement</span>
                    </div>
                    <div class="requirement-content">
                        <p>Une évaluation des fournisseurs et prestataires de services doit être faite régulièrement pour identifier et contrôler les risques. Un enregistrement de toutes les revues et actions entreprises en conséquence de l'évaluation doit être conservé.</p>
                         <div class="tabs">
                            <div class="tab active" onclick="changeTab(this, 'questions-4-4-6')">Questions d'audit</div>
                            <div class="tab" onclick="changeTab(this, 'advice-4-4-6')">Conseils pour les auditeurs</div>
                        </div>
                         <div class="tab-content active" id="questions-4-4-6">
                             <ul class="question-list">
                                <li>À quelle fréquence les informations individuelles de l'évaluation fournisseur sont-elles résumées et examinées ?</li>
                                <li>Quelles conséquences résultent de l'évaluation fournisseur (par exemple, statut) ? Quelles actions résultent d'une évaluation négative ?</li>
                                <li>Comment le courtier informe-t-il les fournisseurs sur les résultats de l'évaluation ? Les fournisseurs sont-ils bloqués, le cas échéant ? Comment le blocage est-il effectué ?</li>
                             </ul>
                        </div>
                        <div class="tab-content" id="advice-4-4-6">
                            <p>L'évaluation fournisseur devrait être effectuée au moins une fois par an. Dans le secteur des fruits et légumes, les fournisseurs sont souvent évalués à la fin de la saison.</p>
                        </div>
                    </div>
                 </div>
                <div class="requirement">
                     <div class="requirement-header" onclick="toggleRequirement(this)">
                        <span class="requirement-number">4.4.7</span>
                        <span class="requirement-title">Les produits achetés doivent être vérifiés conformément aux spécifications et à leur authenticité</span>
                    </div>
                    <div class="requirement-content">
                        <p>Les produits achetés doivent être vérifiés conformément aux spécifications existantes et, sur la base des risques, à leur authenticité. Le calendrier de ces vérifications doit, au minimum, tenir compte des critères suivants : exigences produit et statut du fournisseur (selon son évaluation).</p>
                        <div class="tabs">
                            <div class="tab active" onclick="changeTab(this, 'questions-4-4-7')">Questions d'audit</div>
                            <div class="tab" onclick="changeTab(this, 'advice-4-4-7')">Conseils pour les auditeurs</div>
                        </div>
                         <div class="tab-content active" id="questions-4-4-7">
                             <ul class="question-list">
                                <li>Comment le courtier vérifie-t-il la qualité des produits fournis ?</li>
                                <li>Existe-t-il des spécifications pour un plan d'échantillonnage ? Le courtier effectue-t-il ses propres tests s'il n'y a pas de spécifications client ?</li>
                                <li>Est-il possible d'identifier facilement l'authenticité des produits échangés ? Comment le courtier vérifie-t-il l'authenticité et est-ce vérifiable ?</li>
                             </ul>
                        </div>
                         <div class="tab-content" id="advice-4-4-7">
                             <p>Dans de nombreuses industries, il est courant que les fournisseurs transmettent les résultats des analyses au courtier. Ces informations sont importantes mais ne devraient pas servir de seule source pour le courtier. (voir aussi analyses aux chapitres 5.2.1 et 5.2.2).</p>
                        </div>
                    </div>
                 </div>
                 <div class="requirement">
                    <div class="requirement-header" onclick="toggleRequirement(this)">
                        <span class="requirement-number">4.4.8</span>
                        <span class="requirement-title">En cas de produits de marque client, un système d'approbation des fournisseurs doit exister</span>
                    </div>
                    <div class="requirement-content">
                        <p>En cas de produits de marque client, un système d'approbation des fournisseurs doit exister pour les fournisseurs de produits, qui est conforme aux exigences client.</p>
                        <div class="tabs">
                            <div class="tab active" onclick="changeTab(this, 'questions-4-4-8')">Questions d'audit</div>
                            <div class="tab" onclick="changeTab(this, 'advice-4-4-8')">Conseils pour les auditeurs</div>
                        </div>
                        <div class="tab-content active" id="questions-4-4-8">
                            <ul class="question-list">
                                <li>Des produits de marque client sont-ils fabriqués ? Si oui, quelles réglementations existent pour cela, par exemple, spécifications, questionnaire produit avec caractère de spécification ?</li>
                                <li>Qui remplit les questionnaires client, par exemple de la vente au détail (courtier ou fabricant) ?</li>
                                <li>Si le courtier remplit le questionnaire client : comment l'achèvement correct de toutes les sous-questions du questionnaire client est-il assuré ?</li>
                             </ul>
                        </div>
                        <div class="tab-content" id="advice-4-4-8">
                             <p>Si des produits de marque client sont fabriqués, le courtier doit généralement divulguer le nom du fabricant au client. Comme les courtiers sont des partenaires contractuels, ils reçoivent toutes les spécifications, telles que les questionnaires largement utilisés incluant des détails sur le processus de fabrication (par exemple, détection de métaux, question concernant les risques allergènes, certifications, etc.). Si le courtier répond aux questions du client (par exemple, parce qu'il épargne les frais de traduction ou soulage le fabricant), la question se pose de savoir comment le courtier peut répondre correctement à toutes les questions. Cet aspect est très important et pour les produits de marque client, il est de la plus haute importance de vérifier cela !</p>
                        </div>
                    </div>
                 </div>
            </div>
             <div class="section" id="section4-5">
                 <div class="section-header">
                    <h2>4.5 Emballage du produit</h2>
                    <p>Cette section traite des exigences relatives aux matériaux d'emballage utilisés pour les produits commercialisés.</p>
                 </div>
                <div class="requirement">
                     <div class="requirement-header" onclick="toggleRequirement(this)">
                        <span class="requirement-number">4.5.1</span>
                        <span class="requirement-title">L'entreprise doit s'assurer que des spécifications détaillées existent pour tous les matériaux d'emballage</span>
                    </div>
                    <div class="requirement-content">
                         <p>L'entreprise doit s'assurer que pour les produits importés, les produits de marque propre ou client, des spécifications détaillées existent pour tous les matériaux d'emballage qui pourraient avoir une influence sur le produit. Elles doivent être conformes à la législation applicable du ou des pays de destination du produit.</p>
                        <div class="tabs">
                            <div class="tab active" onclick="changeTab(this, 'questions-4-5-1')">Questions d'audit</div>

                        </div>

                            <ul class="question-list">
                                <li>Des produits sont-ils fabriqués pour le compte du courtier, par exemple des plats préparés, des légumes surgelés ?</li>
                                <li>Des marchandises sont-elles emballées pour le compte du courtier, par exemple des fruits ou des légumes ?</li>
                                <li>Des produits pré-emballés sont-ils importés, par exemple du vin, des spiritueux, des aliments emballés pour le consommateur final ?</li>
                                <li>Des spécifications pour les matériaux d'emballage (primaire et - si applicable - secondaire) des cas mentionnés ci-dessus sont-elles disponibles ?</li>
                                <li>Quelles dispositions légales pour l'emballage sont utilisées ?</li>
                             </ul>
                        </div>
                         <div class="tab-content" id="advice-4-5-1">
                            <p>Veuillez vérifier qui est responsable du produit en fonction du type de produit.</p>
                            <p>Si des produits standardisés emballés et étiquetés sont commercialisés, tels que des articles vendus dans les magasins de détail (appelés "marques A"), les courtiers n'ont qu'une responsabilité limitée pour le produit concernant, par exemple, la manipulation correcte du produit et la conformité avec la durée de conservation restante. Ils ne sont cependant pas responsables de l'emballage, ainsi l'exigence d'existence de spécifications n'est pas applicable.</p>
                            <p>Si les produits sont fabriqués et/ou spécifiquement emballés pour le compte du courtier ou du client ou sont importés de l'extérieur de l'UE par le courtier, le courtier doit présenter des spécifications et des déclarations de conformité.</p>
                        </div>

                     <div class="requirement-header" onclick="toggleRequirement(this)">

                        <span class="requirement-title">Si le matériau d'emballage pourrait compromettre la sécurité du produit, une déclaration de conformité doit être fournie</span>
                    </div>
                    <div class="requirement-content">
                        <p>Si le matériau d'emballage pourrait compromettre la sécurité du produit acheté, une déclaration de conformité doit être fournie par les fournisseurs confirmant la conformité aux exigences législatives. Dans le cas où aucune exigence légale spécifique n'est applicable, des preuves doivent être disponibles pour démontrer que le matériau d'emballage est adapté à l'usage.</p>

                        </div>

                                <li>Des produits sont-ils fabriqués pour le compte du courtier, par exemple des plats préparés, de la volaille surgelée ?</li>
                                <li>Des marchandises sont-elles emballées pour le compte du courtier, par exemple des fruits ou des légumes ?</li>
                                <li>Des produits pré-emballés sont-ils importés, par exemple du vin, des spiritueux, des aliments emballés pour le consommateur final ?</li>
                                <li>Des certificats de conformité pour les emballages plastiques et des déclarations d'innocuité pour d'autres matériaux d'emballage pour les cas mentionnés ci-dessus concernant le matériau d'emballage sont-ils disponibles ?</li>
                             </ul>
                        </div>
                        <div class="tab-content" id="advice-4-5-2">
                            <p>Voir conseil 4.5.1. De plus, il convient de noter que le courtier est responsable de la légalité de l'emballage lors de l'importation dans l'UE, par exemple pour le scellage plastique des vins ou spiritueux importés ou pour les barquettes en plastique pour les fruits ou légumes frais.</p>

                    <div class="requirement-header" onclick="toggleRequirement(this)">

                    </div>
                    <div class="requirement-content">
                         <p>Lorsqu'un changement d'emballage est requis par le client ou la législation, l'entreprise doit s'assurer que l'emballage est contrôlé par le fournisseur et que le produit répond aux exigences légales et/ou client. L'utilisation de l'emballage correct doit être régulièrement vérifiée et les vérifications doivent être documentées par le fournisseur. L'entreprise doit s'assurer que ces vérifications sont entreprises.</p>

                                <li>Comment le courtier s'assure-t-il que le fournisseur/fabricant utilise le bon emballage ? Des spécifications du courtier sont-elles disponibles ?</li>
                                <li>Comment le fabricant vérifie-t-il cela ?</li>
                                <li>Quelles informations le courtier reçoit-il du fabricant ?</li>

Cela ne peut être réalisé qu'en étroite coopération avec le fabricant. Si le fournisseur du courtier est certifié selon IFS ou toute autre norme équivalente reconnue par GFSI, cela devrait être acquis. Ici, les courtiers ont un devoir de diligence et doivent vérifier la conformité aux exigences chez le fabricant, par exemple en ayant les enregistrements appropriés envoyés.</p>

                    </div>
                    <div class="requirement-content">
                         <p>Lorsqu'un changement d'étiquetage produit est requis par le client ou la législation, l'entreprise doit s'assurer que l'étiquetage du produit est modifié par le fournisseur pour répondre aux exigences.                         </div>

                            <p>Pour chaque changement d'étiquetage, qu'il soit initié par les clients ou la législation, le courtier doit s'assurer que les produits échangés sont correctement étiquetés. Cela ne peut être réalisé qu'en étroite coopération avec le fabricant. Si le fournisseur du courtier est certifié selon IFS ou toute autre norme équivalente reconnue par GFSI, cela devrait être acquis. Ici, les courtiers ont un devoir de diligence et doivent vérifier la conformité aux exigences chez le fabricant, par exemple en ayant les enregistrements appropriés envoyés.</p>

                <div class="section-header">
                    <h2>4.6 Traçabilité (y compris OGM et allergènes)</h2>
                    <p>Cette section concerne les systèmes de traçabilité des produits commercialisés.</p>
                </div>
                <div class="requirement">
                    <div class="requirement-header" onclick="toggleRequirement(this)">
                        <span class="requirement-number">4.6.1</span>
                        <span class="requirement-title">Un système de traçabilité doit être en place</span>
                        <span class="ko-badge">KO n°4</span>
                    </div>
                    <div class="requirement-content">
                        <p>Un système de traçabilité doit être en place qui permet l'identification complète des produits. L'étiquetage des produits doit être effectué de manière à permettre une traçabilité complète. Le système de traçabilité et les enregistrements associés doivent garantir une traçabilité complète depuis le fournisseur (défini par lot/quantité) jusqu'à la livraison au client.</p>
                        <div class="tabs">
                            <div class="tab active" onclick="changeTab(this, 'questions-4-6-1')">Questions d'audit</div>
                            <div class="tab" onclick="changeTab(this, 'advice-4-6-1')">Conseils pour les auditeurs</div>
                        </div>
                        <div class="tab-content active" id="questions-4-6-1">
                            <ul class="question-list">
                                <li>Comment les lots commercialisés sont-ils étiquetés ?</li>
                                <li>Quels enregistrements pour une traçabilité complète des produits sont préparés et où sont-ils classés ?</li>
                                <li>Comment l'entreprise informe-t-elle le courtier sur l'étiquetage des lots ou quelles spécifications le courtier donne-t-il pour l'étiquetage des lots ?</li>
                                <li>Quelles quantités contient un lot ?</li>
                                <li>Comment le numéro de lot utilisé est-il composé ?</li>
                            </ul>
                        </div>
                        <div class="tab-content" id="advice-4-6-1">
                            <p>Chez un courtier, le lot pour un article est naturellement défini de la livraison au client. Cela est également valable si le lot commercialisé provient d'un lot uniforme plus important du fabricant, car chaque livraison partielle est soumise à des impacts et à des risques individuels (par exemple, conditions de transport) pendant le transport.</p>
                            <br>
                            <p>Habituellement, le courtier n'a pas de difficultés à différencier les lots puisque des enregistrements sont disponibles pour chaque livraison individuelle (par exemple, commande ou appel au fabricant, documents d'expédition, documentation douanière, si nécessaire, bon de livraison client, facture).</p>
                            <br>
                            <p>Certains fabricants prennent des photos du produit et de l'étiquetage du lot envoyé et les envoient par e-mail au courtier qui est alors informé de l'étiquetage exact grâce aux photos.</p>
                        </div>
                    </div>
                </div>
                <div class="requirement">
                    <div class="requirement-header" onclick="toggleRequirement(this)">
                        <span class="requirement-number">4.6.2</span>
                        <span class="requirement-title">Le système de traçabilité doit être testé régulièrement</span>
                    </div>
                    <div class="requirement-content">
                        <p>Le système de traçabilité doit être testé sur une base régulière - au moins une fois par an et chaque fois que le système de traçabilité change. Le test doit vérifier la traçabilité en amont et en aval (du fournisseur du courtier à son client (y compris les prestataires de services logistiques), et vice versa), y compris la vérification des quantités. Les résultats des tests doivent être enregistrés.</p>

                        </div>

                                <li>L'échantillonnage a été sélectionné selon quels critères ?</li>
                                <li>Des enregistrements complets pour le test sont-ils disponibles ?</li>

                            </ul>

                    </div>
                    <div class="requirement-content">
                        <p>Pour les produits de marque propre et client, le système de traçabilité doit assurer une traçabilité complète depuis la dernière étape de transformation du produit jusqu'à la livraison au client.</p>

                        <span class="requirement-title">Si requis par le client, l'entreprise doit s'assurer que le fournisseur conserve des échantillons représentatifs</span>
                    </div>
                    <div class="requirement-content">
                        <p>Si requis par le client, l'entreprise doit s'assurer que le fournisseur a identifié des échantillons représentatifs pour chaque lot de fabrication, les a stockés de manière appropriée et les a conservés jusqu'à leur date d'expiration ou, si nécessaire, pendant une période plus longue. L'entreprise doit obtenir et conserver une liste de tous les lots fabriqués couverts par les services du courtier.</p>

                <div class="section-header">
                    <h2>4.7 Atténuation de la fraude alimentaire</h2>
                    <p>Cette section concerne les mesures prises pour prévenir la fraude alimentaire tout au long de la chaîne d'approvisionnement.</p>
                    <div class="note"><strong>Note:</strong> ce chapitre est applicable aux produits alimentaires et à leur emballage primaire ainsi qu'au matériel d'emballage destiné aux produits alimentaires. Les produits ménagers et de soins personnels ne sont pas considérés par ce chapitre.</div>
                 </div>

                     <div class="requirement-header" onclick="toggleRequirement(this)">
                        <span class="requirement-number">4.7.1</span>

                    </div>
                    <div class="requirement-content">
                        <p>La responsabilité de l'évaluation de la vulnérabilité à la fraude alimentaire et du plan d'atténuation doit être clairement définie. Les responsables doivent avoir les connaissances spécifiques et l'expertise appropriées, et avoir l'engagement total de la direction.</p>

Les critères pris en compte dans l'évaluation de la vulnérabilité doivent être définis.</p>

                <div class="section-header">
                    <h2>5.2 Analyses produits</h2>
                    <p>Cette section concerne les analyses et contrôles effectués sur les produits pour garantir leur conformité aux spécifications.</p>
                </div>
                <div class="requirement">
                    <div class="requirement-header" onclick="toggleRequirement(this)">
                        <span class="requirement-number">5.2.1</span>
                        <span class="requirement-title">Des procédures d'analyse/test produit doivent garantir la conformité aux exigences</span>
                    </div>
                    <div class="requirement-content">
                        <p>Des procédures d'analyse/test produit doivent être en place pour garantir que toutes les exigences produit spécifiées sont respectées, y compris les exigences légales et les spécifications. Les analyses microbiologiques, physiques et chimiques requises à cette fin doivent être effectuées en interne et/ou sous-traitées.</p>
                        <div class="tabs">
                            <div class="tab active" onclick="changeTab(this, 'questions-5-2-1')">Questions d'audit</div>
                            <div class="tab" onclick="changeTab(this, 'advice-5-2-1')">Conseils pour les auditeurs</div>
                        </div>
                        <div class="tab-content active" id="questions-5-2-1">
                            <ul class="question-list">
                                <li>Comment le courtier vérifie-t-il la conformité aux exigences produit ?</li>
                                <li>Quelles analyses sont requises (microbiologiques, physiques, chimiques) ?</li>
                                <li>Quels paramètres sont examinés ?</li>
                                <li>Quelles exigences client existent concernant les examens requis ?</li>
                            </ul>
                        </div>
                        <div class="tab-content" id="advice-5-2-1">
                             <p>Veuillez vérifier dans quelle mesure le courtier est responsable du produit. En principe, il existe deux situations :</p>
                            <ol>
                                <li>Le courtier commercialise des biens individuels, spécifiquement sélectionnés ou développés pour lui-même ou pour un client (comme les produits de marque propre du courtier ou de marque client) ou importe des produits, par exemple fruits surgelés, viande, fruits et légumes frais, plats préparés fabriqués pour le compte du courtier.</li>
                                <li>Le courtier commercialise des articles standard tels que des produits de marque fournisseur, déjà étiquetés par le fabricant et que le client pourrait également acheter ailleurs, par exemple des produits standardisés qui peuvent être vendus dans les magasins de détail, des matières premières standardisées ou des additifs de grandes entreprises.</li>
                             </ol>
                            <p><strong>Cas 1 :</strong> les courtiers sont entièrement responsables du produit. Ils doivent être capables de démontrer par des tests que le produit a les caractéristiques convenues et est conforme aux exigences légales (car il y a aussi des aspects légaux, par exemple importateur vers l'UE ou non).</p>
                            <p><strong>Cas 2 :</strong> les courtiers sont plutôt des médiateurs. Ils sont responsables des conditions cadres des produits (par exemple, article correct, durée de conservation restante correcte à la livraison, lot uniforme, conformité aux conditions de transport). Ils ne sont pas responsables, cependant, de la composition exacte du produit (par exemple, si une boisson gazeuse caféinée d'une grande entreprise américaine est commercialisée, vous devez contacter le fabricant en cas de réclamations produit et non le courtier, à moins que des aspects sous la responsabilité du courtier ne soient concernés, tels qu'un stockage inapproprié, une durée de vie trop courte, etc.).</p>
                        </div>
                    </div>
                </div>
                <div class="requirement">
                    <div class="requirement-header" onclick="toggleRequirement(this)">
                        <span class="requirement-number">5.2.2</span>
                        <span class="requirement-title">Lorsque des analyses spéciales sont demandées par le client, elles doivent être définies dans un plan d'analyses</span>
                        <span class="ko-badge">KO n°6</span>
                    </div>
                    <div class="requirement-content">
                        <p>Lorsque des analyses spéciales sont demandées par le client, elles doivent être définies dans un plan d'analyses et réalisées conformément aux exigences définies. Les résultats des tests doivent être disponibles sur le site de l'entreprise.</p>
                        <div class="tabs">
                            <div class="tab active" onclick="changeTab(this, 'questions-5-2-2')">Questions d'audit</div>
                            <div class="tab" onclick="changeTab(this, 'advice-5-2-2')">Conseils pour les auditeurs</div>
                        </div>
                        <div class="tab-content active" id="questions-5-2-2">
                            <ul class="question-list">
                                <li>Les produits sont livrés à quels clients ?</li>
                                <li>Le client a-t-il des exigences pour les programmes d'analyse ?</li>
                                <li>Existe-t-il des accords contractuels pour les programmes d'analyse ?</li>
                                <li>Comment la comparaison des quantités est-elle effectuée si des analyses doivent être effectuées selon des quantités déterminées fournies ?</li>
                                <li>Le plan d'inspection comprend-il toutes les analyses spécifiques et autres analyses requises ?</li>
                            </ul>
                        </div>
                        <div class="tab-content" id="advice-5-2-2">
                            <p>Dans le secteur des fruits et légumes, la majorité des clients de la vente au détail ont des exigences liées aux niveaux de résidus des produits de protection des plantes. Ceux-ci permettent, par exemple, seulement 80% des limites légales ou limitent le nombre de substances actives identifiées. Vous devez demander pendant l'audit quels détails spécifiques s'appliquent. L'obligation contractuelle du courtier est généralement déterminée dans des accords-cadres ou des spécifications.</p>
                            <br>
                            <p>Pour le contrôle, des analyses dépendant de la quantité livrée sont convenues (par exemple, une analyse tous les 32 tonnes). Les analyses sont réalisées par des laboratoires agréés, accrédités et saisies dans une base de données de résidus.</p>
                            <br>
                            <p><strong>Notation :</strong> Ici, une notation n'est possible que s'il existe des accords contractuels relatifs aux programmes d'analyse (courant dans le secteur des fruits et légumes, occasionnellement dans d'autres industries). Notation N/A possible s'il n'y a pas de programme spécifique.</p>
                            <p><strong>KO si :</strong> des programmes d'analyse ont été convenus mais il n'y a pas d'outil pour déterminer la date d'échéance/fréquence des analyses ou si l'outil n'inclut pas les analyses demandées.</p>
                            <br>
                            <p>Dans certains cas, les clients informent les fournisseurs de la date des analyses. Dans ce cas, la conformité peut être identifiée par les analyses effectuées et les livraisons. L'outil de contrôle interne pour déterminer le plan d'essais est absent ici, mais les analyses sont effectuées.</p>
                        </div>
                    </div>

                    <div class="requirement-header" onclick="toggleRequirement(this)">
                        <span class="requirement-number">5.2.3</span>
                        <span class="requirement-title">Les analyses pertinentes pour la sécurité produit doivent être effectuées par des laboratoires accrédités (ISO 17025)</span>
                    </div>
                    <div class="requirement-content">
                        <p>Les analyses, qui sont pertinentes pour la sécurité du produit, doivent de préférence être effectuées par des laboratoires ayant des programmes/méthodes accrédités appropriés (ISO 17025). Si les analyses sont effectuées par un laboratoire interne ou un laboratoire n'ayant pas de programmes/méthodes accrédités appropriés, les résultats doivent être vérifiés sur une base régulière par des laboratoires accrédités selon ces programmes/méthodes (ISO 17025).</p>
                         <div class="tabs">
                            <div class="tab active" onclick="changeTab(this, 'questions-5-2-3')">Questions d'audit</div>

                                <li>Quels laboratoires sont mandatés ? Sont-ils accrédités ?</li>
                                <li>Le courtier effectue-t-il ses propres analyses ?</li>

                             <ul class="question-list">
                                <li>L'analyse des dangers reflète-t-elle les différents types de produits ?</li>
                                <li>Quels dangers produit sont identifiés pour les produits échangés ?</li>
                                <li>Quels tests sont déterminés dans le plan de test ?</li>
                                <li>Les paramètres déterminés sont-ils adaptés pour vérifier et valider la conformité des produits échangés ?</li>
                                <li>Le plan de test inclut-il tous les paramètres et produits requis ?</li>
                                <li>Des enregistrements des résultats de test sont-ils disponibles ?</li>

                     }
                 }


                if (matchFound) {
                    let section = req.closest('.section');
                    let sectionId = section?.id || '';
                    resultsFound.push({
                        number: number,
                        title: title, // Titre original pour l'affichage principal
                        preview: previewText, // Aperçu avec highlight potentiel
                        sectionId: sectionId,
                        requirementElement: req // Référence à l'élément pour scroll
                    });
                }
            });

            // Afficher les résultats
            if (resultsFound.length > 0) {
                 // Limiter le nombre de résultats affichés si nécessaire
                 resultsFound.slice(0, 15).forEach(result => {
                    const resultItem = document.createElement('div');
                    resultItem.className = 'search-result-item';
                    // Afficher numéro, titre et aperçu si différent du titre
                    resultItem.innerHTML = `<strong>${result.number}</strong>: ${result.title}`;
                    if (result.preview && result.preview !== result.title) {
                         resultItem.innerHTML += `<br><small style="color: grey;">${result.preview}</small>`;
                    }


                    resultItem.addEventListener('click', function() {
                        if (result.sectionId) {
                            showSection(result.sectionId); // Affiche la bonne section et gère le menu

                            // S'assurer que l'exigence est visible et la faire défiler
                            setTimeout(() => { // Léger délai pour s'assurer que la section est affichée
                                const reqHeader = result.requirementElement.querySelector('.requirement-header');
                                const reqContent = result.requirementElement.querySelector('.requirement-content');

                                // Ouvrir l'accordéon si fermé
                                if (reqContent && !reqContent.classList.contains('active')) {
                                    toggleRequirement(reqHeader);
                                }

                                 // Mettre en évidence le terme dans l'exigence (optionnel, peut être complexe)
                                // highlightTermInElement(result.requirementElement, searchTerm);

                                // Défiler vers l'élément
                                result.requirementElement.scrollIntoView({ behavior: 'smooth', block: 'center' }); // 'center' pour mieux voir

                                 // Masquer les résultats
                                searchResults.style.display = 'none';
                                searchInput.value = '';
                             }, 150); // Augmenter le délai si nécessaire
                        }
                    });
                    searchResults.appendChild(resultItem);
                });
                searchResults.style.display = 'block';
            } else {
                searchResults.innerHTML = '<div class="search-result-item">Aucun résultat trouvé</div>';
                searchResults.style.display = 'block';
            }
        });


        // Masquer les résultats de recherche en cliquant ailleurs
        document.addEventListener('click', function(event) {
            if (!event.target.closest('.search-container')) {
                searchResults.style.display = 'none';
            }
        });

        // Bouton retour en haut
        const backToTopButton = document.querySelector('.back-to-top');

        window.addEventListener('scroll', function() {
            if (window.pageYOffset > 300) {
                backToTopButton.classList.add('visible');
            } else {
                backToTopButton.classList.remove('visible');
            }
        });

        function scrollToTop() {
            window.scrollTo({
                top: 0,
                behavior: 'smooth'
            });
        }

        // Au chargement de la page, gérer le hash de l'URL
        document.addEventListener('DOMContentLoaded', function() {
            const hash = window.location.hash.substring(1);
            if (hash) {
                // Essayer d'abord de trouver une section
                const targetSection = document.getElementById(hash);
                 if (targetSection && targetSection.classList.contains('section')) {
                     showSection(hash);
                 } else {
                     // Essayer de trouver une exigence par son numéro (ex: #req-5.7.2)
                     const reqNumMatch = hash.match(/^req-(\d+\.\d+\.\d+)$/);
                     if (reqNumMatch) {
                        const reqNum = reqNumMatch[1];
                         const reqHeader = Array.from(document.querySelectorAll('.requirement-number'))
                                               .find(el => el.textContent === reqNum)?.closest('.requirement-header');
                         if (reqHeader) {
                             const section = reqHeader.closest('.section');
                             if (section) {
                                 showSection(section.id);
                                 setTimeout(() => {
                                     const reqContent = reqHeader.nextElementSibling;
                                     if (reqContent && !reqContent.classList.contains('active')) {
                                         toggleRequirement(reqHeader);
                                     }
                                     reqHeader.scrollIntoView({ behavior: 'smooth', block: 'center' });
                                 }, 150);
                             }
                         } else {
                            showSection('home-section'); // Fallback
                         }
                     } else {
                        showSection('home-section'); // Fallback si hash non reconnu
                     }
                 }

            } else {
                showSection('home-section'); // Afficher l'accueil par défaut
            }
        });
    </script>
</body>
